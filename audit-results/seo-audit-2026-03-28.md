## Full Execution Plan

### P1: CRITICAL -- Fix before anything else

#### 1. Fix robots.txt -- HTML page appended (D5)

**Issue:** The `robots.txt` file is 53,209 bytes / 90 lines. Lines 1-57 contain valid directives (Content-Signal preamble + User-agent rules). Lines 58-90 contain **the entire minified HTML page** appended after the directives.

**Evidence:** Lighthouse flags `robots-txt` as invalid (score: 0). Last 500 bytes of `/robots.txt` contain `</main> <script type="module">...` HTML content. Crawlers will parse HTML as invalid directives.

**Fix:** Check Astro build config. The static output is likely concatenating the robots.txt with the page output. Ensure `robots.txt` is in `public/` directory (not generated via a page route), or if using `@astrojs/sitemap` integration, verify it's not injecting HTML into the robots file. Verify with `curl -s https://zuvo.dev/robots.txt | tail -5` after deploy.

- SEO Impact: HIGH | Business Impact: HIGH | Effort: EASY
- **Priority: 3.0**

#### 2. Add sitemap.xml (D4) -- CG1 FAIL

**Issue:** `/sitemap.xml` returns the homepage HTML (soft 404). No sitemap exists.

**Fix:** Install `@astrojs/sitemap` integration:
```bash
npx astro add sitemap
```
Add to `astro.config.mjs`:
```js
import sitemap from '@astrojs/sitemap';
export default defineConfig({
  site: 'https://zuvo.dev',
  integrations: [sitemap()],
});
```

- SEO Impact: HIGH | Business Impact: HIGH | Effort: EASY
- **Priority: 3.0**

#### 3. Fix broken OG image (D2)

**Issue:** `og:image` points to `https://zuvo.dev/og.png` which returns 404. Social sharing cards will show no image on Twitter, LinkedIn, Slack, Discord.

**Evidence:** `new Image().src = '/og.png'` fires `onerror`. Image does not exist.

**Fix:** Create an OG image (1200x630px) and place in `public/og.png`. Alternatively, generate dynamically with `@vercel/og` or Astro's image tools.

- SEO Impact: HIGH | Business Impact: HIGH | Effort: EASY
- **Priority: 3.0**

#### 4. Add JSON-LD structured data (D3) -- CG5 FAIL

**Issue:** Zero `<script type="application/ld+json">` elements. Search engines cannot extract structured info. AI engines cannot parse product details.

**Fix:** Add to the Astro layout `<head>`:
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "Zuvo",
  "description": "33 AI development skills for Claude Code with multi-agent pipelines and quality gates.",
  "url": "https://zuvo.dev",
  "applicationCategory": "DeveloperApplication",
  "operatingSystem": "macOS, Linux, Windows",
  "offers": {
    "@type": "Offer",
    "price": "0",
    "priceCurrency": "USD"
  },
  "author": {
    "@type": "Organization",
    "name": "Zuvo",
    "url": "https://zuvo.dev"
  }
}
</script>
```

Also add `WebSite` schema for site-level info and `BreadcrumbList` if subpages are added later.

- SEO Impact: HIGH | Business Impact: HIGH | Effort: EASY
- **Priority: 3.0**
