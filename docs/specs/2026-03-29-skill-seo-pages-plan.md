# Implementation Plan: Skill SEO Pages

**Spec:** docs/specs/2026-03-29-skill-seo-pages-spec.md
**Created:** 2026-03-29
**Tasks:** 10
**Estimated complexity:** 1 standard (schema) + 7 standard (YAML batches) + 1 standard (validation script) + 1 standard (final validation)

## Architecture Summary

- New directory `website/skills/` — zero existing files modified
- 33 YAML data files + 1 schema reference file = 34 files total
- Each YAML generated by reading the corresponding `skills/{slug}/SKILL.md`
- No production code, no APIs, no traditional test files
- Website repo (zuvo.dev) consumes these files at build time — out of scope

## Technical Decisions

- **3 content patterns:** Rich Audit (15 skills), Pipeline/Agent (10 skills), Thin Utility (8 skills)
- **6 batches of 5-6 skills**, grouped by category for content consistency
- **Generation order:** Pipeline-first (most cross-referenced) → Audits → Testing → Performance → Utility (using-zuvo last)
- **Validation script** replaces traditional tests — checks schema conformance, uniqueness, cross-references, description length
- **Stat claims:** Use exact counts from SKILL.md dimension tables. Soft claims (e.g., "60+ checks") use `+` suffix. Do not fabricate precise numbers.

## Quality Strategy

- **No CQ gates apply** — this is content generation, not production code
- **Cookie-cutter risk** is the primary quality concern — mitigated by batching similar skills together and post-generation uniqueness validation
- **Cross-reference integrity** enforced by explicit 33-slug allow-list (not filesystem discovery)
- **Stat accuracy** spot-checked against SKILL.md during generation
- **Validation script** gates the final commit

### Task 9: Create validation script

**Files:** `scripts/validate-skill-pages.sh`
**Complexity:** standard
**Dependencies:** Tasks 2-8 (needs YAML files to exist)
**Model routing:** Sonnet

- [ ] Write a bash script that validates all 33 YAML files:
  ```bash
  #!/bin/bash
  # Validates website/skills/*.yaml files against spec requirements

  ERRORS=0
  SKILLS_DIR="website/skills"
  EXPECTED_COUNT=39

  # 1. Count files (excluding _schema.yaml)
  ACTUAL=$(ls "$SKILLS_DIR"/*.yaml 2>/dev/null | grep -v _schema | wc -l | tr -d ' ')
  if [ "$ACTUAL" != "$EXPECTED_COUNT" ]; then
    echo "FAIL: Expected $EXPECTED_COUNT YAML files, found $ACTUAL"
    ERRORS=$((ERRORS + 1))
  fi

  # 2. Check each file has required top-level keys
  for f in "$SKILLS_DIR"/*.yaml; do
    [[ "$f" == *"_schema"* ]] && continue
    slug=$(basename "$f" .yaml)

    for key in schema_version last_synced meta stats problem how_it_works \
               examples when_to_use when_not_to_use related_skills faq arguments; do
      if ! grep -q "^${key}:" "$f"; then
        echo "FAIL: $slug.yaml missing required key: $key"
        ERRORS=$((ERRORS + 1))
      fi
    done
  done

  # 3. Check meta.description length (< 155 chars)
  for f in "$SKILLS_DIR"/*.yaml; do
    [[ "$f" == *"_schema"* ]] && continue
    slug=$(basename "$f" .yaml)
    desc=$(grep "  description:" "$f" | head -1 | sed 's/.*description: "//' | sed 's/"$//')
    len=${#desc}
    if [ "$len" -gt 155 ]; then
      echo "FAIL: $slug.yaml meta.description is $len chars (max 155)"
      ERRORS=$((ERRORS + 1))
    fi
  done

  # 4. Check title uniqueness
  titles=$(grep "  title:" "$SKILLS_DIR"/*.yaml | grep -v _schema | sed 's/.*title: //' | sort)
  dupes=$(echo "$titles" | uniq -d)
  if [ -n "$dupes" ]; then
    echo "FAIL: Duplicate titles found: $dupes"
    ERRORS=$((ERRORS + 1))
  fi

  # 5. Check cross-reference slugs are in allow-list
  ALLOW_LIST="api-audit architecture backlog brainstorm build canary ci-audit code-audit db-audit debug deploy dependency-audit design design-review docs env-audit execute fix-tests pentest performance-audit plan presentation receive-review refactor release-docs retro review security-audit seo-audit seo-fix ship structure-audit test-audit tests-performance ui-design-team using-zuvo worktree write-e2e write-tests"

  for f in "$SKILLS_DIR"/*.yaml; do
    [[ "$f" == *"_schema"* ]] && continue
    slug=$(basename "$f" .yaml)
    # Extract related skill slugs
    grep "  - slug:" "$f" | sed 's/.*slug: //' | while read ref_slug; do
      if ! echo "$ALLOW_LIST" | grep -qw "$ref_slug"; then
        echo "FAIL: $slug.yaml references unknown slug: $ref_slug"
        ERRORS=$((ERRORS + 1))
      fi
    done
  done

  # 6. Check last_synced is present and non-empty
  for f in "$SKILLS_DIR"/*.yaml; do
    [[ "$f" == *"_schema"* ]] && continue
    slug=$(basename "$f" .yaml)
    if ! grep -q "^last_synced:" "$f"; then
      echo "FAIL: $slug.yaml missing last_synced field"
      ERRORS=$((ERRORS + 1))
    fi
  done

  # 7. Check array cardinality: faq (min 3), stats (min 3), how_it_works (min 3)
  for f in "$SKILLS_DIR"/*.yaml; do
    [[ "$f" == *"_schema"* ]] && continue
    slug=$(basename "$f" .yaml)
    faq_count=$(grep -c "  - q:" "$f" 2>/dev/null || echo 0)
    if [ "$faq_count" -lt 3 ]; then
      echo "FAIL: $slug.yaml has only $faq_count FAQ entries (min 3)"
      ERRORS=$((ERRORS + 1))
    fi
    stats_count=$(grep -c "  - label:" "$f" 2>/dev/null || echo 0)
    if [ "$stats_count" -lt 3 ]; then
      echo "FAIL: $slug.yaml has only $stats_count stat cards (min 3)"
      ERRORS=$((ERRORS + 1))
    fi
  done

  # 8. Summary
  if [ "$ERRORS" -eq 0 ]; then
    echo "PASS: All $EXPECTED_COUNT skill YAML files validated successfully"
  else
    echo "FAIL: $ERRORS validation errors found"
    exit 1
  fi
  ```
